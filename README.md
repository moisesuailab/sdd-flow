# sdd-cli

Spec Driven Development workflow for AI-assisted development.

Installs a structured **Research → Plan → Implement → Validate** cycle into any project, powered by markdown artifacts and isolated agent sessions.

Works with Claude Code, OpenCode, Cursor, Codex CLI, and any agentic IDE with filesystem access.

---

## Install

```bash
npm install -g sdd-cli
```

---

## Usage

```bash
sdd-cli init      # ou: sdd init
sdd-cli update    # ou: sdd update
sdd-cli help      # ou: sdd help
```

### New or existing project

```bash
cd my-project
sdd init
```

Installs the `agents/` folder with all workflow files and optionally sets up safety hooks for your agent.

### Update workflow (preserves your project data)

```bash
sdd update
```

Updates `AGENTS.md`, `RULES.md`, and all prompts in `agents/prompts/`. Your `PROJECT.md`, `DECISIONS.md`, `skills/`, `specs/`, and `harness/` are never touched.

---

## After `init`

1. Open a new session with your agent
2. Instruct: *"Read and follow `agents/SETUP.md`"*
3. Provide your PRD or describe the project
4. Review the generated files and delete `agents/SETUP.md`

---

## The RPI Cycle

Each feature goes through four isolated phases, each in its own agent session:

| Phase | Prompt file | Produces |
|---|---|---|
| Research | `agents/prompts/rpi-research.md` | `RESEARCH.md` |
| Plan | `agents/prompts/rpi-plan.md` | `SPEC.md` + `TASK.md` + `TEST.md` |
| Implement | `agents/prompts/rpi-implement.md` | Code + updated `TASK.md` |
| Validate | `agents/prompts/rpi-validate.md` | `VALIDATION.md` |

---

## What gets installed

```
AGENTS.md                       ← root agent instructions
agents/
  AGENTS.md                     ← full workflow definition
  RULES.md                      ← mandatory process rules
  PROJECT.md                    ← generated by SETUP (your project)
  DECISIONS.md                  ← architectural decisions log
  SETUP.md                      ← initial setup guide (delete after use)
  prompts/
    rpi-research.md
    rpi-plan.md
    rpi-implement.md
    rpi-validate.md
    conventional-commit.md
    pr-template.md
    skill-call.md
    task-create.md
  skills/
    _template/SKILL.md
  harness/                      ← optional, chosen during init
    claude-code/
    opencode/
```
